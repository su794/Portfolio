Pokedex

OVERVIEW

This is a single page of Pokedex using PokeAPI. All the information is brought here from the API. Screen size 1280px is recommended. As users click the 'Next' Button, they can see more information.

Technical Info:

HTML, CSS, JavaScript, API

Source References:

  • https://pokeapi.co
    PokéAPI
  • https://www.pokemon.com
    Pokédex

PROCESS

This started with an assignment on App development strategy to understand how API works. There was a only pokemon list and the 'Next' Button and I added a pop-up card function.

01

Planning

This was an assignment from the class that we need to build a single page and display content using API. I researched for the API examples and I found pokeAPI which has many types of endpoints I can use. Most pokemon evolve twice meaning there are 3 forms in one pokemon. I chose 18 cards to show 6 types of pokemon with different 3 forms.

02

Set the offset and create a 'Next' Button

There are more than 200 pokemon in the API and bringing all of them at once will take long and be heavy to load. So I added the ‘next’ button to load the next portion-18 cards. In this case, Inside API URLs key-value pairs, I put the variable for the pokemon id and limit number of the cards to display more pokemon upon clicking the ‘next’ button.

03

Click and show pop-up card

After I finished my assignment, I tried to add more functions to it. I added a clickable index number below the picture. When users click that button, a small card pops up and shows the Pokemon's abilities, types, height, and weight. For these, I used the different endpoints and for loop statement to show them multiple abilities and types. I also added 'x' button in the popup card to close it and remove the current info using JavaScript. So next time, the information about the pokemon they click will only appear.
